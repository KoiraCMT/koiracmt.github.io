MariaDB 10.1.16 Galera Cluster on CentOS 7

发表于 2016-08-19   |   分类于 MySQL

这网上折腾了两天了，终于是把这个技术的基本实现搞定了，这里做个简单分享吧！按着操作应该是没有问题的，不过有些地方需要你自己替换，我想开始搞这个技术的，应该也不需要我做太多赘述了吧！

node1

安装MySQL

vim /etc/yum.repos.d/MariaDB.repo

yum install MariaDB-server MariaDB-client MariaDB-compat galera -y

mysql没有启动的话:
service mysql start

初始化配置

mysql_secure_installation

建立cluster用户

GRANT ALL PRIVILEGES ON . TO sst@localhost IDENTIFIED BY ‘1234’;
GRANT ALL PRIVILEGES ON . TO ’sst’@‘192.168.10.%’ IDENTIFIED BY ‘1234’;
flush privileges;

关闭MySQL

service mysql stop

mariadb galera集群配置

vim /etc/my.cnf.d/server.cnf

启动mysql

service mysql start –wsrep-new-cluster
第二台开始service mysql start 就行了。只要有一台在 就不用刚刚启动第一次的命令。

查看集群状态

show status like ‘wsrep%’;

我们可以关注几个关键的参数：

wsrep_connected = on 链接已开启

wsrep_local_index = 1在集群中的索引值

wsrep_cluster_size =3集群中节点的数量

wsrep_incoming_addresses = 192.168.10.133:3306 集群中节点的访问地址

在本地有MySQL的情况下，如何使用MAMP的数据库

发表于 2016-03-16   |   分类于 Mac

由于项目需要用到ecshop,而我们都知道它最多只能支持到PHP5.2的版本，再往高版本的话，各种警告。而无论是Mac自带的PHP和我通过brew安装的PHP版本，都已经比它高很多了。那我们如何解决这个尴尬的局面呢，同时不影响本机的环境。本来我是想通过搭一个虚拟机结局的 但感觉挺麻烦的，又要做一套环境，做过环境的朋友知道，好痛！！！ (╯‵□′)╯︵┻━┻
作为一个懒人呢，突然想起了当年刚拿到Mac那会，各种乱折腾软件，里面有个MAMP的PHP集成环境软件。那么作为一个想要成为NB攻城狮的男人，我当初当然放弃了它，自己找虐选择自己编译一个。而它呢，就是Mac下的phpStudy。可以选择你想要的任何PHP版本。

这款软件呢是收费的，但在国内嘛，大家自己搞定吧！安装完毕之后，如果你跟我一样，也是本地已经安装了MySQL了，那么在start环境之后stop环境，就会发现Mysql报错无法关闭之类的意思，具体说什么忘了，应该有这两种情况：

我的理解应该就是两个MySQL的配置文件出现了错误，本地的MySQL配置文件是/etc/my.conf,MAMP的配置文件则是/Applications/MAMP/tmp/mysql/my.conf，所以处理方式也就很简单，需要使用MAMP的时候，将本地的MySQL关闭，并配置文件/etc/my.conf修改下名字就可以了。当然，你后面要用本地的MySQL了，修改回来就可以了。

接着我就想用Navicat操作MySQL，发现又出了问题，明明端口密码什么的都是正确的，就是无法连接。这里呢，还有一个选项需要我们设置。

勾选Use socket file并填入/Applications/MAMP/tmp/mysql/mysql.sock，这样应该就可以通过Navicat访问MAMP的MySQL了。

使用Bootcamp安装windows8.1

发表于 2015-11-25   |   分类于 Mac

前几天因为项目需要，需要使用windows平台下的一个软件开发，本来想着用Parallels Desktop装个windows7的虚拟机开发就行了，结果虚拟机不能与其他同事通信，而我又不太会调配这些东西，于是果断悲剧了，然后就想到Mac下自带了Bootcamp用它装个双系统就可以搞定了，于是就在机子上折腾了一天，真是把自己坑的不轻啊，百度上面也是有攻略的，但多少写的还是不够详细，这里当个记录分享给大家吧。

我的系统是10.10.4的，所以只能装windows8以上版本。Bootcamp在哪我就不用说了吧，不然你还是别用了(￣_￣ )

1.打开BootCamp后点击继续

2.选取如图所示选项点击继续

3.提前下好一个wdinows8.1的系统镜像，并且准备好一个空的U盘，点击继续

注：将ed2k开头的下载链接复制粘贴到「迅雷」中下载。

4.这里会提醒我们格式化U盘！！！！所有如果U盘有什么重要内容，一定要备份好哦！！！！然后点击继续

5.然后就是默默等待吧，估计要30分钟左右吧,过程中如果出现下载失败的情况，就只能从头再来了

安装过程中出现如下图的话，输入你的用户密码即可

6.等待结束后，出现如下图内容

点击红色方框标记的区域分配Windows硬盘大小

这里分配的大小最好不小于30G,不然后面安装windows的话，会出现硬盘过小，无法安装的情况

分区完毕后，点击安装，默默等待。安装好后机器就会重启,没有重启的话请自己重启吧

6.在这里我就碰到了第一个问题，重启之后，竟然又是打开了Mac的系统，完全没有进入Windows的安装界面啊？！

（╯’ - ‘)╯︵ ┻━┻ （掀桌子） ┬─┬ ノ( ‘ - ‘ノ) {摆好摆好） (╯°Д°)╯︵ ┻━┻(再他妈的掀一次）

这里我当时查了资料之后，是这么处理的:

  • 重启机器
  • 在听到开机声音之后，等大概2-3秒之后，长按option键，进入如下图所示界面(没有进来的话，就再重启找感觉吧
    ( •̀ .̫ •́ )✧)
  • 选择EFI Boot，回车(应该是这个分区，当时忘记拍照了，反正就是在Macintosh HD旁边的那个就对了)

PS:这张图是我安装好系统之后才照的，装好之后就出现Windows那个分区了，没装的时候是没有的

7.等待启动进入安装页面

8.按顺序操作后进入如下界面(原谅我的渣技术和渣画质),选择BOOTCAMP，会看到无法在分区4上面安装Windows,然后点击格式化，跳出弹出点击确认

9.格式化后选择分区4，点击下一步，进入安装界面，等待安装，之后就会重启了，重启之后进入个性化设置，这里就不赘述了

10.进入系统后，会发现无法连接网络，各种驱动基本也是没有的

接下来我们就来安装驱动网卡什么的，炒鸡简单，但是没人提醒话也够折腾了( Ĭ ^ Ĭ ),好吧，你们机智，轻松搞定了

  • command+E打开我的电脑

  • 选择D盘

  • 选择BootCamp文件夹

  • 点击setup安装吧

  • 安装重启之后，网卡就OK啦

11.如果想回到Mac下，重装之后会有如图所示的一个图标，右击选择在在OS X中重新启动就可以了

12.回到Mac下，如果到Windows的话，是介个样子的,打开系统偏好设置

  • 点击启动磁盘

  • 选择BOOTCAMP，点击重新启动，就OK啦

大概东西应该就这么多了，按着这个来，应该不会出什么问题的。如果有朋友知道Parallels Desktop虚拟机怎么跟别的机子通信的话，希望能发邮件告诉我哈，如果你允许的话，我也会转载出来，给大家学习的！
