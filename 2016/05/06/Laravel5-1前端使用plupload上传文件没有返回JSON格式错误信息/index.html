Laravel5.1前端使用plupload上传文件没有返回JSON格式错误信息

发表于 2016-05-06   |   分类于 Laravel

昨天使用plupload插件上传文件的时候，向Laravel提交文件后，希望能获得一些错误信息，但发现不太如愿，前后那个折腾啊，本身又没有可以商量的同事，只能自己各种google,可能也是自己的描述问题吧，并不能找到和自己问题相关答案，不废话了，自己上图了：

后端代码

根据官方文档表单验证所说，当我们在 AJAX 的请求中使用 validate 方法时，Laravel 并不会像传统的提交表单的方式生成一个重定向响应，而是会生成一个包含所有错误验证的 JSON 响应。这个 JSON 响应会发送一个 422 HTTP 状态码。

这里我写的验证规则就是判断request中name为img的字段内容是否是一个图片，如果不是，就报错。


这是我上传了一个非图片文件后返回的信息，看里面的内容，我们发现Laravel将我们的页面返回了回来，也就是说Laravel将我们通过plupload提交的文件，认为是通过传统表单的方式提交的request，因此生成了一个重定向响应。

也就是说plupload没有被Laravel判断为AJAX请求，那我们如何处理呢？其实也不复杂，那让Laravel让它以为我们是AJAX请求不就好了嘛。这里我查看了Laravel是如何判断request是否为AJAX请求的源码:

查看源码我们看出来了，Laravel是通过查看headers中名为X-Requested-With的值是否为XMLHttpRequest来判断是否为AJAX的，而查看plupload文档可以看到headers配置参数，那我们将X-Requested-With写入headers参数中即可：
